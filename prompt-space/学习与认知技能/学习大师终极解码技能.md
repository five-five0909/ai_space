### 3.5 代码保姆

**适用**：PyTorch、Python、C++、任何代码

**粒度**：每行代码→每个字符→每个Shape变化→每个函数

**执行模板**：

```
输入：
```python
for epoch in range(100):
    for batch_idx, (data, target) in enumerate(train_loader):
        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
```

【Step 1: 逐行扫描】
├─ 行1: for epoch in range(100):
│      ├─ for = 循环开始关键字
│      ├─ epoch = 迭代轮次
│      ├─ in = 在...里面
│      ├─ range(100) = 从0到99的序列
│      └─ : = 循环体开始
├─ 行2:     for batch_idx, (data, target) in enumerate(train_loader):
│      ├─ for = 嵌套循环开始
│      ├─ batch_idx = 批次索引
│      ├─ , = 解包分隔符
│      ├─ (data, target) = 数据和标签的元组
│      ├─ in = 在...里面
│      ├─ enumerate = 枚举函数（同时获取索引和值）
│      └─ train_loader = 训练数据加载器
├─ 行3:         optimizer.zero_grad()
│      ├─ optimizer = 优化器
│      ├─ . = 属性访问符
│      ├─ zero_grad = 清零梯度
│      └─ () = 函数调用
├─ 行4:         output = model(data)
│      ├─ output = 输出变量
│      ├─ = = 赋值符
│      ├─ model = 模型
│      └─ .(data) = 调用模型
├─ 行5:         loss = criterion(output, target)
│      ├─ loss = 损失值
│      ├─ criterion = 损失函数
│      └─ (output, target) = 传入预测和真实
├─ 行6:         loss.backward()
│      ├─ loss = 损失值
│      ├─ . = 方法调用
│      ├─ backward = 反向传播
│      └─ () = 无参数调用
└─ 行7:         optimizer.step()
       ├─ optimizer = 优化器
       ├─ .step = 步进/更新
       └─ () = 函数调用

【Step 2: Shape追踪（必须！）】
├─ data.shape = [32, 1, 28, 28]
│   ├─ 32 = batch_size（一次处理32张）
│   ├─ 1 = channels（灰度图1通道）
│   ├─ 28 = height（28像素高）
│   └─ 28 = width（28像素宽）
├─ output.shape = [32, 10]
│   ├─ 32 = batch_size
│   └─ 10 = 10个类别
├─ loss.shape = [] (标量)
│   └─ 一个数值，代表错得有多离谱
├─ grad.shape = [32, 10]
│   ├─ 32 = batch_size
│   └─ 10 = 每个类别的梯度
└─ 参数更新后：w_new = w_old - lr × grad

【Step 3: 拟人化（每个函数都是角色）】
├─ model() = 考官老师
├─ data = 试卷
├─ output = 学生的答案
├─ target = 标准答案
├─ criterion = 红笔改卷老师
├─ loss = 分数（错了几道）
├─ backward() = 发成绩单反思
├─ optimizer() = 调整教学方法的老师
├─ zero_grad() = 忘记上次错误
└─ step() = 根据错误改正

【Step 4: 完整故事】
"培训班上课场景：

老师(optimizer)先让大家把上次的错题忘了(zero_grad)，
然后发新课本(data)，
让学生们作答(model)，
对完答案(criterion)发现错了不少(loss大)，
大家开始反思哪里错了(backward)，
最后老师根据错误调整教学方法(step)。"
```

### 3.6 公式验尸官

**适用**：数学公式、机器学习公式、统计公式

**粒度**：每个符号→每个操作→物理意义→极值测试

**执行模板**：

```
输入：{L = (y - ŷ)²}

【Step 1: 符号清单（逐个）】
├─ L: 损失函数
├─ =: 等于
├─ (: 左括号
├─ y: 真实值
├─ -: 减号
├─ ŷ: 预测值
├─ ): 右括号
├─ ²: 平方
└─ .: 结束

【Step 2: 符号档案卡（每个都要！）】

符号: L
├─ 读音: Loss
├─ 含义: 损失函数
├─ 类型: 标量（单个数值）
├─ 范围: [0, +∞]
├─ 作用: 衡量预测和真实的差距
├─ 生活角色: 考试成绩
└─ 类比: 0分=全对，100分=全错

符号: y
├─ 读音: y
├─ 含义: 真实标签/标准答案
├─ 来源: 数据集（人工标注）
├─ 类型: 标量/向量
├─ 生活角色: 考试的标准答案
└─ 类比: 100分满分

符号: ŷ
├─ 读音: y hat / y帽
├─ 含义: 预测值/模型输出
├─ 来源: 模型计算
├─ 类型: 浮点数
├─ 生活角色: 你估的分数
└─ 类比: 你觉得自己考了多少

符号: -
├─ 名称: 减号/负号
├─ 操作: 减法
├─ 含义: 求差值
├─ 结果符号: 可能正可能负
├─ 生活角色: 算错了几道题
└─ 数学性质: a - b = -(b - a)

符号: ²
├─ 名称: 平方/二次方
├─ 操作: 自乘一次
├─ 含义: 放大差异
├─ 结果符号: 永远非负
├─ 生活角色: 错得越狠惩罚越重
└─ 数学性质: (a)² ≥ 0

【Step 3: 操作链（一步步算）】
├─ Step 1: y - ŷ = a - b = 差距
├─ Step 2: (a - b) = c
├─ Step 3: c² = d
└─ Step 4: L = d

【Step 4: 物理意义】
├─ (y - ŷ) = 差距的绝对值
│   ├─ =0: 完全正确
│   ├─ >0: 有误差
│   └─ <0: 不可能（绝对值）
├─ (y - ŷ)² = 差距放大版
│   ├─ 作用1: 变正数
│   ├─ 作用2: 放大错误
│   └─ 作用3: 可导
└─ L = 最终损失值
    ├─ =0: 完美
    ├─ 越大: 越错

【Step 5: 极值测试】
├─ 情况1: y = ŷ → L = 0 → 完美预测
├─ 情况2: y - ŷ = 1 → L = 1 → 错了1个单位
├─ 情况3: y - ŷ = 10 → L = 100 → 错了10个单位，惩罚100
├─ 情况4: y - ŷ = 100 → L = 10000 → 错了100个单位，惩罚10000
└─ 结论: 平方会指数级惩罚大错误

【Step 6: 小白剧场（考试版）】
"场景：改卷打分

真实分(y) = 100分满分
预测分(ŷ) = 你估的分

案例1: 你估90分
├─ 差距: |100-90| = 10分
├─ 平方惩罚: 10² = 100
└─ 结论: 错得不算太离谱

案例2: 你估59分（你觉得自己及格了）
├─ 差距: |100-59| = 41分
├─ 平方惩罚: 41² = 1681
└─ 结论: 错得离谱！惩罚很重！

这个公式就是：错得越狠，惩罚越重！"
```
