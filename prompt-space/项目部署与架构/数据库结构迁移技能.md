# 🛠️ Claude Code Skill: 数据库迁移与字段同步 (通用版)

**适用场景**：任何 Java/MySQL 项目的 Entity 与数据库字段不一致、SQL 执行错误

---

## 1. 快速开始

**如果你的项目遇到以下错误，立即使用此 Skill：**
- `Unknown column 'xxx' in 'field list'`
- `Column 'xxx' not found` - Entity 字段无法映射
- `Data too long for column` - 字段长度不匹配
- `Incorrect date value` - 日期格式错误
- `Duplicate entry` - 唯一索引冲突

**执行指令：**
> 请执行 **数据库迁移技能**，诊断并修复字段同步问题

## 3. 第二步：表结构对比分析

### 3.1 获取数据库表结构

```sql
-- MySQL 连接
mysql -h ${DB_HOST:localhost} -P ${DB_PORT:3306} -u ${DB_USERNAME:root} -p${DB_PASSWORD:} ${DB_NAME}

-- 查看所有表
SHOW TABLES;

-- 查看单个表结构
DESCRIBE table_name;

-- 或详细结构
SHOW CREATE TABLE table_name;

-- 查看所有字段信息
SELECT TABLE_NAME, COLUMN_NAME, DATA_TYPE, COLUMN_KEY, IS_NULLABLE, COLUMN_DEFAULT, COLUMN_COMMENT
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = '${DB_NAME}'
ORDER BY TABLE_NAME, ORDINAL_POSITION;
```

### 3.2 获取 Entity 结构

```java
// 读取 Entity 类，提取字段信息
// 例如：BusTask.java

@TableName("bus_task")
public class BusTask {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String title;           // 数据库: title (VARCHAR)
    private String description;      // 数据库: description (TEXT)
    private Integer taskType;       // 数据库: task_type (INT)
    private BigDecimal reward;     // 数据库: reward (DECIMAL)
    private LocalDateTime completeTime;  // 数据库: complete_time (DATETIME)
}
```

### 3.3 对比工具

```bash
# 创建对比脚本
cat > compare_schema.sh << 'EOF'
#!/bin/bash

# 参数: 表名
TABLE=$1
DB_NAME=$2

echo "=== 数据库表结构 ==="
mysql -h localhost -u root -p123456 $DB_NAME -e "DESCRIBE $TABLE;" 2>/dev/null

echo ""
echo "=== Entity 字段 ==="
# 提取 Entity 中的字段（需要根据实际项目调整）
grep -E "private|@TableField" src/entity/${TABLE}.java 2>/dev/null
EOF
```

## 4. 第三步：生成迁移脚本

### 4.1 新增字段

```sql
-- 添加新字段
ALTER TABLE table_name
ADD COLUMN column_name VARCHAR(50) COMMENT '字段说明' AFTER existing_column;

-- 添加多个字段
ALTER TABLE table_name
ADD COLUMN field1 VARCHAR(50) COMMENT '字段1',
ADD COLUMN field2 INT DEFAULT 0 COMMENT '字段2',
ADD COLUMN field3 DATETIME COMMENT '字段3';
```

### 4.2 修改字段

```sql
-- 修改字段类型
ALTER TABLE table_name
MODIFY COLUMN column_name VARCHAR(100) COMMENT '修改后的说明';

-- 重命名字段
ALTER TABLE table_name
CHANGE COLUMN old_name new_name VARCHAR(50) COMMENT '新说明';

-- 设置默认值
ALTER TABLE table_name
ALTER COLUMN column_name SET DEFAULT 0;
```

### 4.3 删除字段

```sql
-- 删除字段（谨慎操作，可能丢失数据）
ALTER TABLE table_name
DROP COLUMN if exists column_name;
```

### 4.4 创建索引

```sql
-- 普通索引
ALTER TABLE table_name
ADD INDEX idx_column_name (column_name);

-- 唯一索引
ALTER TABLE table_name
ADD UNIQUE INDEX uk_column_name (column_name);

-- 组合索引
ALTER TABLE table_name
ADD INDEX idx_multi (col1, col2, col3);
```

## 5. 第四步：Entity 注解修复

### 5.1 MyBatis-Plus 注解

```java
// 通用 Entity 模板
package com.yourproject.entity;

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@TableName("table_name")  // 对应数据库表名
public class EntityName {

    @TableId(type = IdType.AUTO)  // 主键
    private Long id;

    // 字段名映射（默认自动转换：userName -> user_name）
    @TableField("custom_field")  // 如果字段名不一致，指定数据库列名
    private String customField;

    // 非数据库字段
    @TableField(exist = false)
    private String extraField;

    // 自动填充字段
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;

    // 逻辑删除
    @TableLogic
    private Integer deleted;
}
```

### 5.2 JPA/Hibernate 注解

```java
// JPA Entity 模板
package com.yourproject.entity;

import jakarta.persistence.*;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@Entity
@Table(name = "table_name")
public class EntityName {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "custom_field", length = 50)  // 指定列名和长度
    private String customField;

    @Column(nullable = false, length = 100)
    private String name;

    @Column(columnDefinition = "TEXT")  // 指定数据库类型
    private String description;

    @CreationTimestamp  // 自动填充创建时间
    private LocalDateTime createTime;

    @UpdateTimestamp  // 自动填充更新时间
    private LocalDateTime updateTime;

    @Transient  // 非持久化字段
    private String extraField;
}
```

### 5.3 驼峰与下划线转换

```yaml
# application.yml
mybatis-plus:
  configuration:
    # 开启驼峰转下划线（默认开启）
    map-underscore-to-camel-case: true
```

**转换规则：**

| Java 字段 | 数据库列 |
|----------|---------|
| userName | user_name |
| taskType | task_type |
| completeTime | complete_time |
| ID | id |

## 6. 第五步：常见问题修复

### 6.1 问题分类速查表

| 错误信息 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `Unknown column 'xxx'` | Entity 有但数据库没有 | 数据库添加字段 |
| `ResultSet has no data` | 数据库有但 Entity 没有 | Entity 添加字段 |
| `Data too long` | 字段长度不匹配 | 扩展字段长度 |
| `Incorrect date value` | 日期格式错误 | 统一日期格式 |
| `Duplicate entry` | 唯一索引冲突 | 处理重复数据 |
| `Lock wait timeout` | 事务锁等待 | 优化事务或增加超时 |

### 6.2 问题修复代码

**问题：字段类型不匹配**

```java
// Java: BigDecimal <-> 数据库: DECIMAL
private BigDecimal price;  // 推荐用于金额

// Java: LocalDateTime <-> 数据库: DATETIME
private LocalDateTime createTime;

// Java: String <-> 数据库: TEXT (长文本)
private String content;
```

**问题：时间戳字段**

```java
// 推荐使用 LocalDateTime
private LocalDateTime createTime;
private LocalDateTime updateTime;

// 如果数据库是 timestamp 自动更新
// 数据库: `update_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
```

**问题：枚举类型映射**

```java
// 方式1：存储枚举的 name
private UserStatus status;  // 存: "ACTIVE", "INACTIVE"

// 方式2：存储枚举的 value
@EnumValue
private Integer statusCode;

// 配置转换器
@Converter
public class StatusConverter implements AttributeConverter<UserStatus, Integer> {
    @Override
    public Integer convertToDatabaseColumn(UserStatus attribute) {
        return attribute.getCode();
    }
    @Override
    public UserStatus convertToEntityAttribute(Integer dbData) {
        return UserStatus.of(dbData);
    }
}
```

## 7. 第六步：迁移脚本管理

### 7.1 创建迁移脚本

```sql
-- 文件: V1__add_complete_time.sql
-- 场景：添加 complete_time 字段到 bus_task 表

-- 1. 添加字段
ALTER TABLE bus_task
ADD COLUMN complete_time DATETIME COMMENT '完成时间' AFTER urgency;

-- 2. 验证字段已添加
DESCRIBE bus_task;

-- 3. 验证数据
SELECT id, title, complete_time FROM bus_task LIMIT 5;
```

### 7.2 Flyway 迁移管理

```xml
<!-- Maven -->
<dependency>
    <groupId>org.flywaydb</groupId>
    <artifactId>flyway-core</artifactId>
    <version>10.0.0</version>
</dependency>
```

```yaml
# application.yml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true
```

**迁移脚本命名规范：**


## 9. 附录：常用 SQL 速查

```sql
-- 查看表状态
SHOW TABLE STATUS LIKE 'table_name';

-- 查看索引
SHOW INDEX FROM table_name;

-- 查看建表语句
SHOW CREATE TABLE table_name;

-- 查看最近修改的表
SELECT TABLE_NAME, UPDATE_TIME
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = 'db_name'
ORDER BY UPDATE_TIME DESC;

-- 检查重复数据
SELECT column_name, COUNT(*) as cnt
FROM table_name
GROUP BY column_name
HAVING cnt > 1;

-- 修复重复数据
DELETE t1 FROM table_name t1
INNER JOIN table_name t2
WHERE t1.id < t2.id AND t1.column_name = t2.column_name;
```

## AI 执行要点

作为 AI，你需要：

1. **先探测后修改**：先读取数据库配置和 Entity，不要假设
2. **根据 ORM 框架**：选择正确的注解（MyBatis-Plus vs JPA）
3. **提供可执行的 SQL**：让用户可以直接在数据库执行
4. **说明风险**：如删除字段、修改类型可能导致数据丢失
5. **提供回滚方案**：确保迁移可逆
