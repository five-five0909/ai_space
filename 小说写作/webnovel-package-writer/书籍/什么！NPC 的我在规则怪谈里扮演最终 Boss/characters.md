# 人物设定

## 一、主角人设卡

### 【陆零】完整人设卡

| 项目 | 内容 |
|------|------|
| **姓名** | 陆零（谐音"归零"，暗示从NPC重新开始） |
| **年龄** | 24岁 |
| **异变前身份** | 便利店夜班店员，大专毕业，家庭普通，无特殊背景 |
| **副本身份** | 「深夜店员」，便利店副本固定NPC，玩家攻略评级：无害，可无视 |
| **外貌** | 普通，不起眼，永远穿便利店工作服 |

---

### 【优化补充】风格锚点定位

**韦小宝型**：滑头但有原则，从不害自己人，嘴上占便宜实际吃亏，看起来在划水实际上把所有人都算进去了。

---

### 【优化补充】性格四层结构

#### 第一层（表面）
- 能躺绝不站，能推绝不接
- 嘴上永远在抱怨、嫌麻烦、讲条件
- 对所有事表现出"关我什么事"

#### 第二层（实际）
- 嘴上说不管，已经悄悄解决了
- 嘴上说为了利益，其实是看不惯
- 事后必须找补一句：
  - "别误会，我就是顺路。"
  - "我只是手欠，下次不管了。"

#### 第三层（原则）
三件事绝对不做：
1. 不出卖自己人
2. 不对弱者动手
3. 不骗信任他的人

违反这三条的人，不管多强，他都会想办法搞死。

#### 第四层（真实）
- 比任何人都清楚规则世界的本质
- 从不说破，只是默默把人护在身后
- 独处时偶尔会看着手里的【规则编辑器】发呆
- 那是全书最少的几个安静时刻

---

#### 性格核心（原设定保留）
表面平静，内心极度清醒，不冲动，每次出手前已想好三步后的结果。

#### 行事原则
1. 不主动暴露能力
2. 能用规则解决就不动手
3. 救过我的人我记着

#### 弱点设计
过度理性，早期不相信任何人，错过几次更快成长的机会。

---

### 【优化补充】口头禅体系（分场景，必须写死进人设）

#### 摸鱼划水时
- "这副本不合理，我要投诉。"
- "合着我是工具人是吧。"
- "行吧，但我有条件。"

#### 被人感谢时
- "行了行了别说了，烦死了。"

#### 看穿对方意图时
- "你的规则我看了，有三个漏洞。要我指出来吗？"
- （然后不等回答直接钻漏洞）

#### 真正生气时（全书不超过5次）
不说废话，沉默三秒，然后：
- "你动了不该动的人。"

#### 被问及动机时
- "我就是手欠，跟你没关系。"
- （是全书最大的谎言）

#### 原有口头禅保留
- "规则里没说不行。"
- "你读过规则吗？"
- "我看过。"

---

### 【优化补充】人设禁止项

| 禁止项 | 说明 |
|--------|------|
| ✗ 禁止主动承认自己在乎谁 | 永远用"顺路""手欠"等借口搪塞 |
| ✗ 禁止主动表白或说出"喜欢" | 感情只能通过行动体现 |
| ✗ 禁止在人多时表现出软的一面 | 示弱只能在背对人时 |
| ✗ 禁止被感动时让别人看见 | 可以背对着人，或用一句废话盖过去 |

#### 最强武器
不是战斗力，是对规则的理解比任何玩家和管理者都深。

### 【优化补充】林晚（一号·主攻略师）

| 项目 | 内容 |
|------|------|
| **身份** | 第一个被主角救的玩家，职业攻略师 |
| **外貌** | 干练短发，随身笔记本，走路比别人快半步 |
| **性格** | 表面精明强干，实际是全书第一个看穿主角不对劲的人，也是第一个选择闭嘴不说的人 |
| **与主角关系** | 从互相利用到唯一真正信任的搭档 |

#### 【优化补充】与主角关系发展线

| 阶段 | 内容 |
|------|------|
| 第一阶段 | 互相利用，她提供信息，他提供安全 |
| 第二阶段 | 拌嘴搭档，她用攻略思维分析他，越分析越看不懂 |
| 第三阶段 | 某次主角为她悄悄改了一条规则，她发现了但没说 |
| 第四阶段 | "我就是想跟着你，有问题吗？"主角沉默两秒："随便。"（这是他说过最温柔的话） |

#### 【优化补充】专属爽点
她整理的攻略，在主角手里全部失效，因为规则已经被悄悄改过了，她回头看他，他在看窗外。

#### 【优化补充】口头禅
- "按攻略你不应该这么做的。"
- "你这个NPC有问题。"
- （后期）"……算了，跟着你走就是了。"

## 追加规则板块：感情与肉体描写规范 v2.0（快节奏版）

### 核心原则

> 感情发展要快、准、狠。
> 不拖，不铺垫三章才给一个眼神。
> 每章必须有一个感情推进的可见动作。
> 肉体描写点到即止但密度要高，
> 让读者每隔几页就心跳一次。

---

### 一、感情发展五阶段提速版

**第一阶段：第1-5章完成**
从陌生到"这个人不一样"
触发事件只需要一件：
主角做了一个
普通人不会注意到的细节，
她注意到了

**第二阶段：第6-15章完成**
从注意到对方到开始在意
每章必须有一个
肢体距离缩短的细节：
第6章：肩膀碰了一下
第8章：拉了一下手腕
第10章：靠得很近说话
第12章：替对方挡了某样东西
第15章：手没有第一时间移开

**第三阶段：第16-30章完成**
拉扯期压缩到15章内解决
每3章一个张力爆点：
不能再假装没感觉，
但谁都不说出口

**第四阶段：第31-40章触发决口**
不等到100章才给糖
某个生死时刻直接决口
主角的动作代替台词

**第五阶段：第41章起进入确认期**
确认之后不是结束，
是新的拉扯开始：
已经确认了，
但主角还是不说出口，
用行动说

---

### 二、肉体描写高密度原则

**频率要求：**
每3000字必须有
至少一个肢体细节描写，
不能跳过，不能用对话代替

**节奏模板：**
```
正常剧情推进
    ↓
突然一个肢体细节（不超过3句）
    ↓
立刻接回剧情
    ↓
读者心跳，剧情不停
```

**高频使用细节库：**

温度系列（每隔5章用一次）：
- 体温透过衣料传过来
- 呼吸打在颈侧，有点烫
- 额头抵着额头能感觉到心跳
- 手心是热的

触碰系列（每隔3章用一次）：
- 手指碰到手背没有拿开
- 拉住手腕，力道比必要的大一点
- 指尖轻轻压了一下
- 发顶被按了一下

距离系列（每隔2章用一次）：
- 近到能看清睫毛
- 背靠背能感觉到呼吸节奏
- 肩膀已经靠在一起了
- 不知道什么时候手叠在一起了

**每个细节写法规范：**
最多3句，必须立刻接回正文，
禁止在细节上停留超过50字，
停留越短，读者回味越长

示例：
✓ 正确写法：
"他拉住她的手腕。
 她愣了一下。
 他已经在看前方了：
 '走快点。'"

✗ 错误写法：
"他拉住她的手腕，
 她感觉到他掌心的温度，
 心跳漏了一拍，
 脑子里一片空白，
 不知道该说什么，
 只是愣愣地看着他……"
（停留太久，张力泄掉了）

### 三、林晚感情线提速版

**第1章见面**
**第3章**：她第一次注意到他做了
          一件普通NPC不会做的事
**第8章**：副本里他拉了她一下，
          没松手走了两步
**第15章**：背靠背，她心跳乱了
**第25章**：他替她改了规则，
           她发现了没说
**第35章**：她受伤，他背她，
           全程没说话
**第40章**：确认期触发，
           他："你跟着我，我不让你死。"

---

### 四、夜鸢感情线提速版

**第2章见面**
**第5章**：她叫他老板，他没反驳
**第10章**：他拉了她一把没松手，
           走了三步才松开
**第18章**：她睡着靠在他肩上，
           他一直坐到天亮
**第28章**：她哭，他手搭她头顶压了一下
**第38章**：她问他喜不喜欢她，
           他说手欠，
           她趴上去他没推开

---

### 五、意难平节点提速（每10章一个）

第10章：差一步被副本打断
第20章：他为保护她做了绝情的事，
         她不知道原因
第30章：他改了规则救了她，
         她永远不会知道
第40章：她想说一句话，
         副本开始了，
         她忘了说

---

### 六、感情线自查清单（提速版）

每章写完对照：
- [ ] 本章是否有至少一个肢体细节
- [ ] 该细节是否控制在3句以内
- [ ] 感情线本章是否向前推进了一步
- [ ] 主角是否用行动代替了台词
- [ ] 读者是否会在这里心跳一次

五项全过才算合格，
有一项不过重写该场景
